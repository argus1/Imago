A Software Bill of Materials (SBOM) is a formal, machine-readable "list of ingredients" that details the components, libraries, and supply chain relationships used to build a software application. Guidance documents **establish the minimum data fields and formats required for software transparency, vulnerability management, and supply chain security**. \[[1](https://www.nist.gov/itl/executive-order-14028-improving-nations-cybersecurity/software-supply-chain-security-guidance-20#:~:text=The%20National%20Telecommunications%20and%20Information%20Administration%20\(NTIA\),Confirming%20that%20SBOMs%20meet%20the%20NTIA's%20Recom), [2](https://media.defense.gov/2025/Sep/03/2003791481/-1/-1/0/JOINT-GUIDANCE-A-SHARED-VISION-OF-SOFTWARE-BILL-OF-MATERIALS-FOR-CYBERSECURITY.PDF#:~:text=An%20SBOM%20is%20a%20formal%20record%20of,be%20thought%20of%20as%20a%20%E2%80%9Clist%20o), [3](https://www.wiz.io/academy/application-security/standard-sbom-formats#:~:text=What%20are%20popular%20SBOM%20formats?%20Three%20formats,the%20SBOM%20ecosystem:%20Software%20Package%20Data%20Exchan)\]

**Key SBOM Guidance Frameworks**

* **NTIA Minimum Elements:** The foundational U.S. baseline for what data must be included in an SBOM, such as supplier name, component name, version, and unique identifier.  
* **CISA Guidelines:** Updates the baseline through 2025 Minimum Elements for a Software Bill of Materials (or newer versions), outlining best practices for software transparency and risk management.  
* **International Consensus:** Joint guidance on a Shared Vision of Software Bill of Materials for Cybersecurity aligns global standards across borders to simplify compliance.  
* **Federal Directives:** U.S. requirements under Executive Order 14028 dictate that software vendors selling to the U.S. government must provide SBOMs. \[[5](https://sbomify.com/compliance/#:~:text=Executive%20Order%2014028%20\(2021\)%20%2D%20The%20binding,Memorandum%20M%2D26%2D05%20\(January%202026\)%20rescinded%20the%20mand), [9](https://security.cms.gov/posts/new-cybersecurity-guidance-cisa-software-bill-materials-sbom#:~:text=On%20August%2022%2C%202025%2C%20the%20Cybersecurity%20&,Bill%20of%20Materials%20\(SBOM\).%20The%20draft%2C%202025)\]

**Required Elements of an SBOM**

* **Supplier:** The name of the entity that creates and distributes the component.  
* **Component Name:** The designation assigned to the software product.  
* **Version:** The specific release or version number of the component.  
* **Unique Identifier:** A string or hash that consistently and uniquely identifies the component (e.g., purl or CPE).  
* **Dependency Relationship:** Data showing how components connect and rely on one another.  
* **Author:** The person or tool generating the SBOM data.  
* **Timestamp:** The date and time the SBOM document was created. \[[4](https://www.ntia.gov/page/software-bill-materials#:~:text=A%20collection%20of%20initiatives%2C%20guidance%2C%20models%2C%20frameworks%2C,the%20value%20of%20SBOM.%20Software%20Identity:%20Challenges), [10](https://www.ibm.com/think/topics/sbom#:~:text=Supplier%20name:%20The%20entity%20that%20creates%2C%20defines%20and%20manufactures%20the%20software%20component), [11](https://www.accountablehq.com/post/healthcare-sbom-requirements-practical-compliance-guide-for-providers-and-medical-device-manufacturers#:~:text=Unique%20identifiers%20\(e.g.%2C%20PURL%2C%20CPE%2C%20or%20SWID\)%2C%20plus%20cryptographic%20hashes%20for%20integrity.), [12](https://certera.com/blog/ultimate-guide-to-sboms-software-bill-of-materials/#:~:text=Unique%20Identifier:%20Every%20component%20in%20the%20SBOM,ID%20number%20in%20addition%20to%20its%20hash.), [13](https://www.ntia.gov/sites/default/files/publications/automating_compliance_tooling_-_2021.06.17_0.pdf#:~:text=Unique%20Identifier%20A%20string%20that%20uniquely%20identifies,a%20Globally%20Unique%20Identifier?%20It%20would%20b)\]

**Standard Machine-Readable Formats**

* **SPDX:** The Software Package Data Exchange format acts as a standard for managing licensing and compliance across supply chains.  
* **CycloneDX:** A lightweight, highly adopted standard developed specifically for application security context and vulnerability tracking. \[[1](https://www.nist.gov/itl/executive-order-14028-improving-nations-cybersecurity/software-supply-chain-security-guidance-20#:~:text=The%20National%20Telecommunications%20and%20Information%20Administration%20\(NTIA\),Confirming%20that%20SBOMs%20meet%20the%20NTIA's%20Recom), [3](https://www.wiz.io/academy/application-security/standard-sbom-formats#:~:text=What%20are%20popular%20SBOM%20formats?%20Three%20formats,the%20SBOM%20ecosystem:%20Software%20Package%20Data%20Exchan), [14](https://www.sonatype.com/blog/what-are-sbom-standards-and-formats#:~:text=SBOM%20Formats:%20A%20Language%20for%20SBOMs.%20SBOM,the%20details%20of%20software%20components%20within%20a), [15](https://notes.kodekloud.com/docs/Certified-Kubernetes-Security-Specialist-CKS/Supply-Chain-Security/SBOM-Format/page#:~:text=CycloneDX%20Format%20CycloneDX%20is%20designed%20as%20a,emphasis%20on%20security%20and%20compliance.%20It%20i), [16](https://www.ox.security/blog/sbom-tools/#:~:text=CycloneDX:%20Originating%20from%20the%20OWASP%20community%2C%20CycloneDX,was%20developed%20to%20integrate%20with%20application%20securi)\]

**Why SBOMs Are Used**

* **Vulnerability Tracking:** Quickly identifies if your software contains known, compromised components.  
* **License Compliance:** Ensures open-source licenses match your company's usage policies.  
* **Risk Management:** Maps out the exact origin of third-party and open-source code within your products. \[[2](https://media.defense.gov/2025/Sep/03/2003791481/-1/-1/0/JOINT-GUIDANCE-A-SHARED-VISION-OF-SOFTWARE-BILL-OF-MATERIALS-FOR-CYBERSECURITY.PDF#:~:text=An%20SBOM%20is%20a%20formal%20record%20of,be%20thought%20of%20as%20a%20%E2%80%9Clist%20o), [8](https://www.cisa.gov/resources-tools/resources/shared-vision-software-bill-materials-sbom-cybersecurity#:~:text=CISA%20and%20the%20National%20Security%20Agency%20\(NSA\),have%20released%20joint%20guidance%20outlining%20a%20shared), [17](https://www.lineaje.com/post/whats-in-an-sbom-and-why-it-matters-for-risk-managers#:~:text=Compliance:%20Many%20companies%20use%20open%20source%20software%2C,ensures%20that%20you're%20following%20these%20rules%2C%20avoi), [18](https://panorays.com/glossary/what-are-software-bill-of-materials-sbom-in-tpcrm/#:~:text=Licensing%20Details:%20Documents%20open%2Dsource%20licenses%20and%20usage,ensure%20compliance%20with%20legal%20and%20regulatory%20requirements.)\]
